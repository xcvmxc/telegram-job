# Telegram Job Scanner

[English](README.md) · **Русский**

Скилл для ИИ-агентов-кодеров, который просматривает Telegram-каналы, **на
которые вы уже подписаны**, оставляет только вакансии, подходящие под ваш запрос,
и складывает их в аккуратный Markdown-файл — по одной команде: `/tg-intent`.

Отбор делает сам агент, прямо внутри вашей сессии.
**Не нужен ни платный AI-ключ, ни сервер** — единственное, что потребуется, это
бесплатный личный API-ключ Telegram.

**Работает с:** Claude Code · OpenAI Codex · Gemini CLI · Cursor.
**Языки:** английский (по умолчанию) или русский — выбирается при установке.

---

## Что он делает

```
/tg-intent
  → читает список каналов             (Telegram Sources.md)
  → тянет новые посты с прошлого запуска (курсор на канал, лишнего не качает)
  → по каждому посту агент решает: это реальная вакансия? подходит вам?
  → сохраняет подходящие               (вакансии+2026-07-13_1430.md)
```

Вы управляете двумя обычными текстовыми файлами (на выбранном языке):

| Файл | Для чего |
|------|----------|
| `Search Criteria.md`  | Что вы ищете, простыми словами. Правите — и `/tg-intent` отбирает по-новому. |
| `Telegram Sources.md` | Какие каналы/группы сканировать, по одному в строке. |

Бэкенд сканера лежит в общей, независимой от агента папке (`~/.tgjobs`), поэтому
все агенты, куда вы его поставите, используют одни и те же каналы, критерии и
историю.

## Что нужно

- Хотя бы один из: **Claude Code**, **OpenAI Codex**, **Gemini CLI**, **Cursor**.
- **[uv](https://astral.sh/uv)** — запускает библиотеку Telegram в изолированном
  окружении. Установка: `curl -LsSf https://astral.sh/uv/install.sh | sh`
- **Аккаунт Telegram**, состоящий в каналах, которые вы хотите сканировать.

## Установка

**Одна команда, без клонирования.** Спросит язык и каких агентов настроить,
затем поставит:

```bash
curl -fsSL https://raw.githubusercontent.com/xcvmxc/telegram-intent/main/install.sh | bash
```

Хотите без вопросов? Передайте флаги (повторный запуск в любой момент добавляет
ещё агента):

```bash
curl -fsSL .../install.sh | bash -s -- --lang ru --agent claude,codex
# --lang en|ru   --agent claude|codex|gemini|cursor|all (через запятую)
```

<details><summary>Из клона</summary>

```bash
git clone https://github.com/xcvmxc/telegram-intent.git && cd telegram-intent
./install.sh                 # интерактивно
./install.sh --lang ru --agent all
```
</details>

Установщик не трогает ваши состояние (`~/.tgjobs/jobs/jobs.db`) и конфиг и делает
резервную копию любого конфига агента, который меняет. Он же настраивает каждого
агента выполнять пайплайн **без запроса подтверждения** — allow-list для Claude
Code / Gemini / Cursor, а для **Codex** спрашивает, прежде чем вписать песочницу
+ `approval_policy = never` в `~/.codex/config.toml` (это же даёт `/tg-intent` выход
в сеть и запись вне папки проекта — песочница Codex, так задумано).

Затем в агенте запустите **`/tg-intent-setup`** — мастер проведёт через:

1. **API-ключ Telegram** — бесплатно, ~1 мин на [my.telegram.org](https://my.telegram.org) → *API development tools*.
2. **Вход** — разовый вход в Telegram.
3. **Рабочая папка** — где хранить файлы и результаты; два редактируемых файла
   кладутся туда на выбранном языке.

## Как пользоваться

1. Отредактируйте **`Search Criteria.md`** — опишите нужные роли.
2. Отредактируйте **`Telegram Sources.md`** — добавьте каналы (по одному в
   строке). Для **приватных** каналов сначала вступите по инвайт-ссылке, затем
   добавьте её. Список всех каналов, где состоит ваш аккаунт:
   ```bash
   uv run --with telethon python ~/.tgjobs/telegram/tg_scan.py list
   ```
3. Запустите **`/tg-intent`**. Откройте файл `вакансии+...md`, который он создаст.

Запускайте `/tg-intent` когда угодно — он смотрит только посты новее прошлого
запуска (из любого агента), поэтому повторы дешёвые и без дубликатов.

**Чтобы поменять, что искать:** правьте `Search Criteria.md`. **Источники:**
правьте `Telegram Sources.md`. Больше ничего — без повторной настройки.

**Дубли ролей:** вакансия не пишется повторно, если та же **компания +
должность** уже встречалась за последние дни — даже под другой ссылкой из
другого канала. Окно настраивается через `"export_dedup_days"` в `config.json`
(по умолчанию `2`, `0` — выключить; ограничено `retention_days`). Точные дубли
по ссылке отсекаются всегда.

**Ссылки и посты:** `/tg-intent` ищет и по apply-ссылкам, **и** по текстовым постам
целиком — вакансия без ссылки попадает как сам пост (с короткой выдержкой).
**Уборка:** сообщения и вакансии старше **2 дней** удаляются в начале каждого
сканирования (настройка `"retention_days"` в `config.json`; она же ограничивает
окно дедупа выше).

## Язык

Выберите английский или русский при установке. Это задаёт язык диалога,
формулировки двух редактируемых файлов и формулировки выходного файла. Чтобы
переключить позже — перезапустите установщик с другим `--lang`.

## Обновление

`/tg-intent` в конце прогона проверяет наличие новой версии (не чаще раза в сутки) и
**предлагает** обновиться — нужно только подтвердить. Одно обновление освежает
общий бэкенд и **все агенты**, куда установлен скилл, сразу, сохраняя всё ваше
состояние. Можно обновить и вручную:

```bash
curl -fsSL https://raw.githubusercontent.com/xcvmxc/telegram-intent/main/install.sh | bash -s -- --update
```

## Из чего это собрано

```
~/.tgjobs/                          общий, независимый от агента бэкенд
  jobs/{config,db,scan,setup,update}.py  конвейер (stdlib Python)
  jobs/jobs.db  jobs/config.json    состояние + конфиг (не перезаписываются)
  jobs/templates/{en,ru}/           разворачиваемые файлы, по языку
  telegram/tg_scan.py               фетчер на Telethon
  telegram/credentials.env          ваши TG_API_ID / TG_API_HASH
  telegram/jobscan.session          ваша сессия входа в Telegram

на каждого агента (тонкий адаптер → указывает на ~/.tgjobs):
  Claude Code   ~/.claude/commands/tg-intent{,-setup}.md
  Codex         ~/.agents/skills/tg-intent{,-setup}/SKILL.md (+ ~/.codex/skills/)
  Gemini CLI    ~/.gemini/commands/tg-intent{,-setup}.toml   (+ settings.json)
  Cursor        ~/.cursor/skills/tg-intent{,-setup}/SKILL.md (+ permissions.json)
```

## Про правила Telegram

Инструмент читает каналы через ваш собственный аккаунт (через
[Telethon](https://docs.telethon.dev/)) — тот же контент, что вы и так видите в
приложении. Автоматизация пользовательского аккаунта — «серая зона» по правилам
Telegram, и при агрессивном использовании аккаунт могут ограничить или
заблокировать. Сканируйте в человеческом темпе, только каналы, где вы состоите,
и на свой риск.

## Удаление

```bash
rm -rf ~/.tgjobs \
       ~/.claude/commands/tg-intent*.md \
       ~/.agents/skills/tg-intent* ~/.codex/skills/tg-intent* \
       ~/.gemini/commands/tg-intent*.toml \
       ~/.cursor/skills/tg-intent*
```

Изменённые конфиги агентов (`~/.gemini/settings.json`, `~/.cursor/permissions.json`,
`~/.codex/config.toml`) сохраняют бэкапы `.tgjobs.bak` — верните их руками, если
хотите убрать записи из allowlist. Рабочую папку скрипт не трогает.
